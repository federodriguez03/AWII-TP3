TP 2: API-REST

OBTENER PRODUCTOS

Método: GET
Obtiene un recurso, en este caso todos los productos de ‘productos.json’.
Ruta: localhost:3000/v1/productos

{
    "productos": [
        {
            "id": "1",
            "nombre": "Smartphone Galaxy S24+",
            "marca": "Samsung",
            "categoria": "Smartphone",
            "stock": "50"
        },
        {
            "id": "2",
            "nombre": "Laptop XPS 13",
            "marca": "Dell",
            "categoria": "Laptop",
            "stock": 30
        },
        {
            "id": "3",
            "nombre": "Auriculares WH-1000XM4",
            "marca": "Sony",
            "categoria": "Auriculares",
            "stock": 100
        }
    ]
}

En caso de error:

  • Código 500 "Internal Server Error”
    • Descripción: indica que el servidor encontró una condición inesperada que le impidió cumplir con la solicitud del cliente.
    • Mensaje: ‘Error de servidor’

Si los datos fueron enviados:

  • Código 200 "OK”
    • Descripción: indica que la solicitud del cliente ha sido recibida, procesada y completada exitosamente por el servidor.

Si no se encuentra la ruta:

  • Código 404 "Not Found”
    • Descripción: indica que el servidor no puede encontrar el recurso solicitado.
    • Mensaje: ‘Ruta no encontrada’

CREAR PRODUCTO

Método: POST
Crea un nuevo recurso, y lo almacena en un archivo JSON, en ‘productos.json’.

Ruta: localhost:3000/v1/productos

{
    "productos": [
        {
            "nombre": "Smartphone Samsung Galaxy A34",
            "marca": "Samsung",
            "categoria": "Smartphone",
            "stock": "50"
        }
    ]
}

// El ID se genera automáticamente y de forma enumerada.

En caso de error:

  • Código 500 "Internal Server Error”
    • Descripción: indica que el servidor encontró una condición inesperada que le impidió cumplir con la solicitud del cliente.
    • Mensaje: ‘Error de servidor’

Si los datos fueron enviados:

  • Código 201 "Creado”
    • Descripción: indica que la solicitud ha tenido éxito y que se ha creado un nuevo recurso como resultado de la misma.
    • Mensajes: 'Archivo insertado’, 'Recurso creado correctamente’

Si no se encuentra la ruta:

  • Código 404 "Not Found”
    • Descripción: indica que el servidor no puede encontrar el recurso solicitado.
    • Mensaje: ‘Ruta no encontrada’

MODIFICAR PRODUCTO

Método: PUT
Actualiza un recurso completamente, y lo almacena en un archivo JSON, en ‘productos.json’.

Ruta: localhost:3000/v1/productos{id}

// Antes de modificar 
{
    "productos": [
						{
							"id": 11,
							"nombre": "Smartphone Galaxy A34",
							"marca": "Samsung",
							"categoria": "Smartphone",
							"stock": "10"
						}
			]
}
//Después de modificar
{
    "productos": [
						{
							"id": 11,
							"nombre": "Smartphone Galaxy A54",
							"marca": "Samsung",
							"categoria": "Smartphone",
							"stock": "5"
						}
			]
}

En caso de error:

  • Código 500 "Internal Server Error”
    • Descripción: indica que el servidor encontró una condición inesperada que le impidió cumplir con la solicitud del cliente.
    • Mensajes: 'Error al recibir los datos en el servidor’, 'Error al escribir en el archivo.’, 'Error al escribir en el archivo.’

Si los datos fueron enviados:

  • Código 200 "OK”
    • Descripción: indica que la solicitud del cliente ha sido recibida, procesada y completada exitosamente por el servidor.
    • Mensajes: 'Producto modificado con éxito.’

Si no se encuentra la ruta:

  • Código 404 "Not Found”
    • Descripción: indica que el servidor no puede encontrar el recurso solicitado.
    • Mensajes: 'Producto no encontrado.’, ‘Ruta no encontrada’
